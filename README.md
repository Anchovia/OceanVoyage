# OceanVoyage

OceanVoyage는 Vulkan 기반 자체 엔진으로 개발하는 실험적 게임 프로젝트입니다.

현재 이 저장소는 Pastel-Farm 프로젝트에서 복제된 초기 전환 상태입니다.
따라서 아직 농장 게임 코드, 에셋, 셰이더, 문서가 상당 부분 남아 있습니다.

장기 목표는 이 프로젝트를 **싱글플레이 산업혁명 시대 해운·무역·산업 성장 RPG**로 전환하는 것입니다.

## 현재 상태

* 렌더링 백엔드: Vulkan
* 코드베이스 출처: Pastel-Farm
* 현재 게임플레이 상태: 농장 게임 프로토타입 코드가 남아 있음
* 해상/항해 게임플레이: 아직 구현되지 않음
* 목표 방향: 싱글플레이 산업혁명 시대 해운·무역 RPG
* 온라인/MMO: 초기 버전에서는 계획하지 않음

## 목표 게임 방향

OceanVoyage는 1700년대부터 1900년대 사이, 특히 1830년대부터 1880년대 사이의 산업혁명 전환기를 주요 분위기로 삼는 해운·무역·산업 성장 게임을 목표로 합니다.

플레이어는 작은 배 한 척을 가진 선장으로 시작해, 점차 선주, 무역회사 운영자, 해운회사 사장, 산업 자본가로 성장합니다.

핵심 테마는 다음과 같습니다.

* 항해
* 해상 무역
* 화물 운송
* 항로 개척
* 항구 성장
* 범선과 증기선의 공존
* 석탄, 철광석, 강철, 기계부품 같은 산업 자원
* 회사 성장
* 경제적 진행

## 현재 개발 목표

해상 게임플레이를 구현하기 전에, 먼저 Pastel-Farm에서 가져온 엔진 기반을 정리하고 안정화합니다.

우선 목표는 다음과 같습니다.

* 프로젝트 정체성 정리
* 재사용 가능한 엔진 코드와 농장 게임 전용 코드 분리
* Vulkan 리소스 처리 개선
* 렌더링 기본 품질 개선
* 전환 과정에서도 항상 빌드 가능한 상태 유지
* 대체 테스트 씬이 생기기 전까지 농장 시스템을 무작정 삭제하지 않기

성능 기준: RTX 3060급 / 1080p~1440p / 60fps를 비주얼·렌더링 작업의 기준으로 삼는다. 사실적 해양 스택(FFT/Gerstner 바다, 화면공간 반사, 고해상 CSM 그림자, 선박 PBR 등)을 감당하기 위한 기준이며, 이전 GTX 1660 Super 기준은 폐기한다.

## 렌더링 제1원칙

OceanVoyage의 모든 개발과 구현은 **RTX 3060급 하드웨어에서 AAA/AA급 사실적 해양 게임을 만든다**는 원칙을 최우선으로 한다.

- 저사양 기준, 임시 가짜 효과, 화면에 선을 그리는 식의 wake/foam, 저해상도 노이즈 블롭, flat color stand-in을 최종 품질처럼 구현하지 않는다.
- 물, 선박, 그림자, 반사, 거품, 항적, 대기 표현은 실제 상용 게임과 엔진에서 쓰이는 안정적인 기법을 우선한다.
- 선택 기준은 "가장 쉬운 코드"가 아니라 **안정성, 확장성, Vulkan 정공법, 최종 렌더러로 성장 가능한 구조**다.
- placeholder가 필요하면 명시적으로 placeholder라고 표시하고, 최종 기법을 막지 않는 범위에서만 둔다.
- wake/foam 같은 고품질 효과는 별도 mask/texture, advection/decay, simulation/update pass 등 실제 게임에서 쓰는 기반으로 접근한다. 단순 셰이더 도색으로 대체하지 않는다.

## 전환 원칙

기존 코드를 무작정 삭제하지 않습니다.

각 시스템은 먼저 다음 네 가지 중 하나로 분류합니다.

1. 재사용 가능한 엔진 코드로 유지
2. 참고용으로 임시 유지
3. 농장 게임 전용 코드로 판단해 나중에 제거
4. OceanVoyage 전용 시스템으로 교체

각 단계가 끝날 때마다 프로젝트는 빌드 가능한 상태여야 합니다.
