# OceanVoyage

OceanVoyage는 Vulkan 기반 자체 엔진으로 개발하는 실험적 게임 프로젝트입니다.

이 저장소는 Pastel-Farm 프로젝트에서 복제되어 OceanVoyage로 전환 중입니다.
**렌더링 스택은 사실적 해양 방향으로 크게 전환됐고**(아래 "현재 상태" 참고), 게임플레이·월드·UI·저장 계층에는 아직 농장 게임 구조가 남아 있습니다.

장기 목표는 이 프로젝트를 **싱글플레이 산업혁명 시대 해운·무역·산업 성장 RPG**로 전환하는 것입니다.

## 현재 상태

렌더링 백엔드는 Vulkan이며, 사실적 해양 렌더 스택이 다음 수준까지 구현되어 있습니다.

**구현된 렌더링**

* 해수면: 다중 캐스케이드 **Tessendorf FFT** 바다 (GPU 컴퓨트: 스펙트럼 → 분산관계 애니메이션 → butterfly IFFT → displacement/slope 조립)
* 선박 항적: world-locked **wake 시뮬레이션 마스크** (이류·확산·감쇠, Kelvin arm, 선체 프로파일 주입) — 화면 도색이 아님
* 반사: **화면공간 반사(SSR, temporal reprojection)** + 플래너 반사 + 분석적 하늘 폴백
* 선박: **PBR(Cook-Torrance GGX)** 머티리얼 (albedo/normal/specular 맵, DDS 로드)
* 그림자: **CSM(3 캐스케이드)** + Poisson PCF 소프트 섀도
* 거품: FFT Jacobian 기반 whitecap + wake foam (절차 노이즈 블롭 아님)
* 후처리: **HDR(R16F) + ACES 톤매핑** + 컬러 그레이딩, **FXAA / SMAA**
* 부력: FFT displacement CPU 리드백으로 선박을 실제 파면 위에 부유·기울임
* 하늘/대기: 시간대(낮/밤·달) 연동 분석적 하늘, 장거리 안개

**미구현 게임플레이**

* 해상/항해 게임플레이(항해 물리·풍향·경제·항구·교역): **아직 없음**
* 현재 이동은 **농장 캐릭터 컨트롤러 잔재**(복셀 타일 격자 충돌 이동)이며, 항해 물리로 교체 예정
* 기본 월드 생성은 물 타일 기반 해상 테스트 상태지만, `World`/`Chunk`/`TileType`/인벤토리/핫바/저장 포맷은 아직 농장 구조를 사용
* 온라인/MMO: 초기 버전에서는 계획하지 않음

> 요약: **"화질 기준점"은 사실적 해양 스택으로 섰고, 다음 본작업은 선박 상태·항해 물리·렌더 데이터 경계를 OceanVoyage 전용 구조로 교체하는 것**입니다.

## 목표 게임 방향

OceanVoyage는 1700년대부터 1900년대 사이, 특히 1830년대부터 1880년대 사이의 산업혁명 전환기를 주요 분위기로 삼는 해운·무역·산업 성장 게임을 목표로 합니다.

플레이어는 작은 배 한 척을 가진 선장으로 시작해, 점차 선주, 무역회사 운영자, 해운회사 사장, 산업 자본가로 성장합니다.

핵심 테마는 다음과 같습니다.

* 항해
* 해상 무역
* 화물 운송
* 항로 개척
* 항구 성장
* 범선과 증기선의 공존
* 석탄, 철광석, 강철, 기계부품 같은 산업 자원
* 회사 성장
* 경제적 진행

## 현재 개발 목표

사실적 해양 렌더 스택(화질 기준점)은 섰으므로, 다음 단계는 **그 위에 항해 게임플레이를 얹고 농장 잔재를 걷어내는 것**입니다.

전체 개발 순서(Phase 1~10)는 `docs/ROADMAP.md`에 정리돼 있습니다. 첫 본작업은 **Phase 1 — 기본 선박 상태와 항해 물리**입니다.

우선 목표는 다음과 같습니다.

* 항해 이동 모델 도입(농장 타일-워크 컨트롤러 → 관성·풍향 기반 선박 이동)
* `ShipState`/`ShipController` 또는 동등한 항해 상태 모델 도입
* 렌더러의 `World`/`Chunk`/`TileType` 직접 의존 축소(`FrameRenderData`를 순수 렌더 스냅샷으로 정리)
* 농장 전용 시스템(작물·농기구·제작·타일 상호작용·핫바/인벤토리 UI·세이브 필드) 단계적 제거
* 렌더링 후속 과제(TAA 도입, 셰이더 상수 단일화, async compute 검토, 부력 리드백 축소, 플래너/SSR 비용 정책 — `docs/ENGINE_TODO.md`)
* 전환 과정에서도 항상 빌드 가능한 상태 유지

성능 기준: RTX 3060급 / 1080p~1440p / 60fps를 비주얼·렌더링 작업의 기준으로 삼는다. 사실적 해양 스택(다중 캐스케이드 FFT 바다, SSR/플래너 반사, 고해상 CSM 그림자, 선박 PBR 등)을 감당하기 위한 기준이며, 이전 GTX 1660 Super 기준은 폐기한다.

## 렌더링 제1원칙

OceanVoyage의 모든 개발과 구현은 **RTX 3060급 하드웨어에서 AAA/AA급 사실적 해양 게임을 만든다**는 원칙을 최우선으로 한다.

- 저사양 기준, 임시 가짜 효과, 화면에 선을 그리는 식의 wake/foam, 저해상도 노이즈 블롭, flat color stand-in을 최종 품질처럼 구현하지 않는다.
- 물, 선박, 그림자, 반사, 거품, 항적, 대기 표현은 실제 상용 게임과 엔진에서 쓰이는 안정적인 기법을 우선한다.
- 선택 기준은 "가장 쉬운 코드"가 아니라 **안정성, 확장성, Vulkan 정공법, 최종 렌더러로 성장 가능한 구조**다.
- placeholder가 필요하면 명시적으로 placeholder라고 표시하고, 최종 기법을 막지 않는 범위에서만 둔다.
- wake/foam 같은 고품질 효과는 별도 mask/texture, advection/decay, simulation/update pass 등 실제 게임에서 쓰는 기반으로 접근한다. 단순 셰이더 도색으로 대체하지 않는다.

## 전환 원칙

기존 코드를 무작정 삭제하지 않습니다.

각 시스템은 먼저 다음 네 가지 중 하나로 분류합니다.

1. 재사용 가능한 엔진 코드로 유지
2. 참고용으로 임시 유지
3. 농장 게임 전용 코드로 판단해 나중에 제거
4. OceanVoyage 전용 시스템으로 교체

각 단계가 끝날 때마다 프로젝트는 빌드 가능한 상태여야 합니다.
