# OceanVoyage 전환 계획

이 문서는 Pastel-Farm에서 복제된 현재 코드베이스를 OceanVoyage로 전환하기 위한 계획을 정리한다.

목표는 농장 게임 코드를 즉시 전부 삭제하는 것이 아니다.
재사용 가능한 엔진 코드는 보존하고, 농장 게임 전용 시스템은 안전하게 제거하며, 전환 과정에서도 프로젝트가 계속 빌드 가능한 상태를 유지하는 것이 목표다.

## Phase 0: 프로젝트 정체성 정리

상태: 시작 전

작업 목록:

* 프로젝트 이름과 문서에서 Pastel-Farm 흔적 정리
* README 갱신
* 전환 계획 문서 추가
* 엔진 구조 문서 추가
* 프로젝트가 계속 빌드되는지 확인
* 아직 주요 게임플레이 시스템은 삭제하지 않음

## Phase 1: 엔진 / 게임 경계 검토

상태: 시작 전

기존 코드를 네 가지 그룹으로 분류한다.

### 1. 유지

OceanVoyage에서도 재사용 가능성이 높은 시스템.

예시:

* Vulkan 렌더러
* 윈도우 / 입력 처리
* 카메라 기반 코드
* 텍스처 로딩
* 셰이더 컴파일 흐름
* 디버그 UI
* GPU 프로파일링
* FrameRenderData 구조
* 저장 / 로드 기반 구조
* 기본 오브젝트 렌더링
* 후처리
* FXAA / SMAA

### 2. 임시 유지

농장 게임 전용 성격이 있지만, OceanVoyage 시스템을 만들 때 참고 가치가 있는 코드.

예시:

* World / Chunk 구조
* 인벤토리 로직
* 오브젝트 배치 로직
* UI 생성 방식
* 저장 포맷 처리
* 타일 상호작용 로직

### 3. 나중에 제거

Pastel-Farm 전용 게임플레이에 직접 묶여 있는 시스템.

예시:

* 작물 성장
* 농기구
* 밭 / 물주기 규칙
* 작업대 제작
* 울타리 배치 규칙
* 농장 전용 아이템 정의
* 농장 전용 오브젝트 정의

### 4. 교체

OceanVoyage에 맞는 새 시스템으로 바꿔야 하는 영역.

예시:

* 타일 월드 → 해상 / 섬 / 항구 / 지역 월드 시스템
* 플레이어 이동 → 선박 / 선장 / 회사 조작
* 농장 인벤토리 → 화물창 / 회사 창고
* 작물 경제 → 교역품 / 자원 / 산업 경제
* 농장 오브젝트 → 선박, 항구, 건물, 화물, 산업 시설

## Phase 2: 최소 테스트 씬

상태: 시작 전

농장 게임플레이를 제거하기 전에, 먼저 최소한의 OceanVoyage 테스트 씬을 만든다.

테스트 씬 목표:

* Vulkan 창 생성
* 자유 카메라 또는 궤도 카메라
* 하늘 / 안개 / 후처리 유지
* 임시 물 평면
* 임시 선박 오브젝트 1개
* 디버그 UI
* 정상 종료

이 테스트 씬이 빌드되고 실행되기 전에는 대규모 농장 시스템 삭제를 하지 않는다.

## Phase 3: 농장 시스템 제거

상태: 시작 전

농장 전용 시스템은 작은 커밋 단위로 제거한다.

권장 제거 순서:

1. 작물 게임플레이
2. 농기구
3. 제작 / 작업대 규칙
4. 농장 전용 아이템 정의
5. 농장 전용 오브젝트 정의
6. 농장 전용 타일 상호작용
7. 농장 전용 UI 패널

각 제거 단계 후에는 반드시 빌드 가능 상태를 유지한다.

## Phase 4: OceanVoyage 초기 프로토타입

상태: 시작 전

초기 OceanVoyage 프로토타입 목표:

* 조작 가능한 선박 1척
* 간단한 바다 평면 또는 타일형 수면
* 선박 추적 / 궤도 카메라
* 기본 이동
* 임시 항구 1개
* 단순 화물 구매 / 판매 루프
* 플레이어 위치와 화물 저장 / 로드
