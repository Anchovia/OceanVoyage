# 실행 상태 점검표

이 문서는 OceanVoyage 전환 작업 중 프로젝트가 계속 정상 실행되는지 확인하기 위한 수동 점검표다.

현재 저장소는 Pastel-Farm에서 복제된 뒤 OceanVoyage로 전환 중이다. 해양 렌더링 기준점은 구현됐지만, 게임플레이·월드·UI·저장 계층에는 아직 농장 게임 구조가 남아 있다.

따라서 이 점검표는 “현재 기준으로 정상 동작하는지”를 확인하기 위한 임시 기준이다.

이 문서는 자동 테스트를 대체하지 않는다.

다만 코드 삭제, 이름 변경, 렌더러 정리, GameState 교체 전후에 최소한의 회귀를 확인하기 위해 사용한다.

---

## 1. 기본 빌드 확인

### Debug 빌드

```bash
cmake -B build -S .
cmake --build build --config Debug
```

확인 항목:

- [ ] CMake configure가 성공한다.
- [ ] shader compile 단계가 성공한다.
- [ ] C++ 빌드가 성공한다.
- [ ] 실행 파일이 생성된다.
- [ ] 실행 파일 옆에 `shaders/` 폴더가 복사된다.
- [ ] 실행 파일 옆에 `assets/` 폴더가 복사된다.

---

## 2. 클린 빌드 확인

기존 build 폴더를 지운 뒤 다시 빌드한다.

PowerShell:

```powershell
Remove-Item -Recurse -Force build
cmake -B build -S .
cmake --build build --config Debug
```

Git Bash:

```bash
rm -rf build
cmake -B build -S .
cmake --build build --config Debug
```

확인 항목:

- [ ] clean configure가 성공한다.
- [ ] clean build가 성공한다.
- [ ] FetchContent 의존성 구성이 성공한다.
- [ ] Vulkan SDK의 `glslc`를 찾는다.
- [ ] 모든 shader가 `.spv`로 컴파일된다.

---

## 3. 실행 확인

확인 항목:

- [ ] 프로그램이 실행된다.
- [ ] 창이 생성된다.
- [ ] 첫 화면이 지나치게 오래 흰 화면으로 머물지 않는다.
- [ ] 게임 화면이 표시된다.
- [ ] 마우스 / 키보드 입력이 반응한다.
- [ ] 프레임이 계속 갱신된다.
- [ ] 창 닫기가 정상 동작한다.
- [ ] 종료 시 크래시가 발생하지 않는다.

---

## 4. 기본 렌더링 확인

확인 항목:

- [ ] 메뉴/로딩/게임 화면 전환이 표시된다.
- [ ] 해상 테스트 월드가 표시된다.
- [ ] 선박이 표시된다.
- [ ] 오브젝트가 비정상적으로 검게 보이지 않는다.
- [ ] 텍스처가 지나치게 흐리거나 깨지지 않는다.
- [ ] 그림자가 표시된다.
- [ ] 그림자가 심하게 깜빡이지 않는다.
- [ ] 하늘 / 안개 / 시간대 표현이 정상적으로 보인다.
- [ ] 후처리가 적용된다.
- [ ] UI가 화면 위에 표시된다.

---

## 4.1 해양 렌더링 확인

확인 항목:

- [ ] 해수면이 FFT 파도로 움직인다(평면이 아니라 다중 스케일 파도).
- [ ] 수평선까지 바다가 이어지고 원거리에서 깨지거나 타일 반복이 두드러지지 않는다.
- [ ] 햇빛 윤슬/스펙큘러가 과하게 번쩍이거나(over-bright) shimmer가 심하지 않다.
- [ ] 반사(하늘·선박)가 수면에 보이고, grazing 각도에서 프레넬 반사가 강해진다.
- [ ] 선박이 파면 위에 떠서 파도에 따라 높이·기울기가 바뀐다(공중에 뜨거나 잠기지 않음).
- [ ] 선박을 움직이면 항적(wake) 거품/자국이 생기고 시간이 지나면 사라진다.
- [ ] 선박 PBR 머티리얼이 빛 방향에 반응한다(낮/밤·달 포함).
- [ ] 선박 그림자(CSM)가 보이고 캐스케이드 경계에서 튀지 않는다.
- [ ] 메뉴/로딩 화면에서는 바다 컴퓨트가 돌지 않는다(불필요한 부하 없음).

---

## 5. 입력 확인

확인 항목:

- [ ] 이동 키가 동작한다. 현재는 항해 물리가 아니라 농장 컨트롤러 잔재라는 점을 감안한다.
- [ ] 카메라 회전 / 줌 입력이 동작한다.
- [ ] 마우스 입력이 UI와 충돌하지 않는다.
- [ ] UI 조작이 동작한다.
- [ ] 창 focus를 잃었다가 돌아와도 입력이 이상해지지 않는다.

---

## 6. 저장 / 로드 확인

확인 항목:

- [ ] 저장 명령이 동작한다.
- [ ] 저장 파일이 생성된다.
- [ ] 저장 후 게임을 종료해도 크래시가 없다.
- [ ] 재실행 시 저장 데이터가 로드된다.
- [ ] 현재 구조 기준으로 플레이어/선박 위치가 복원된다.
- [ ] 현재 구조 기준으로 인벤토리 상태가 복원된다(화물 시스템 도입 전 임시 필드).
- [ ] 현재 구조 기준으로 바닥 드롭 아이템이 복원된다(제거 예정 필드).
- [ ] 수정된 월드 / 청크 상태가 복원된다(제거 또는 Voyage save schema 교체 예정).
- [ ] 잘못된 저장 파일이 있을 때 프로그램이 즉시 크래시하지 않는다.

---

## 7. 청크 / 월드 확인

확인 항목:

- [ ] 시작 위치 주변 청크가 로드된다.
- [ ] 이동하면 새 청크가 로드된다.
- [ ] 멀어진 청크가 정리된다.
- [ ] 청크 경계에서 눈에 띄는 깨짐이 없다.
- [ ] 저장 후 다시 불러와도 수정된 청크 상태가 유지된다.

---

## 8. 개발 UI 확인

개발 빌드 옵션이 켜져 있을 때 확인한다.

확인 항목:

- [ ] Dev UI가 표시된다.
- [ ] FPS 또는 frame time 정보가 표시된다.
- [ ] GPU profiling 정보가 표시된다.
- [ ] AA 옵션이 변경된다.
- [ ] 디버그 옵션 변경 후 크래시가 없다.

---

## 9. 전환 작업 후 필수 확인

다음 작업 이후에는 반드시 이 점검표를 다시 확인한다.

- 프로젝트 이름 변경
- CMake 설정 변경
- shader 목록 변경
- renderer 초기화 코드 변경
- VulkanContext 생성 / 파괴 코드 변경
- GameState 변경
- ShipState / 항해 물리 변경
- World / Chunk 변경
- save/load 변경
- 농장 시스템 제거
- 렌더러와 World/Chunk 의존성 분리

---

## 9.1 ROADMAP Phase별 점검 (도입 시 활성)

아래 항목은 해당 `docs/ROADMAP.md` Phase가 구현되면 활성화한다. 지금은 미구현이므로 점검 대상이 아니다.

**Phase 1 — 항해 물리**
- [ ] W 가속 → 떼면 관성 후 감속(즉시 정지 아님)
- [ ] W+A / W+D가 곡선 선회(즉각 90° 꺾임 아님)
- [ ] 정지 중 A/D만으로 제자리 회전하지 않음
- [ ] 선회·이동 중에도 선박이 파면에 계속 떠 있고 wake가 heading/velocity를 따라감

**Phase 2 — 농장 제거 / HUD**
- [ ] 화면에 hotbar/inventory/crafting 농장 UI가 보이지 않음
- [ ] 선박 HUD(SPD/HDG/THR/RUD)가 보임
- [ ] 메뉴/설정/로딩 UI는 정상 동작

**Phase 3 — VoyageSave / 교역**
- [ ] 재실행 후 선박 위치·heading·money·cargo 복원
- [ ] 손상 save에서 즉시 크래시하지 않음, 구 PFRM save를 잘못 읽지 않음
- [ ] 항구 근처에서 매매 가능, 항구 2개 간 교역으로 이익 발생

**Phase 4 이후 — 렌더/항해 심화**
- [ ] (TAA) 선박 edge/wake 끌림·윤슬 shimmer 감소, 카메라 회전 시 과한 번짐 없음
- [ ] (풍향/돛) 풍각에 따라 같은 throttle에서도 속도 변화, no-go zone 경고
- [ ] (항구/섬) 섬 충돌로 통과 불가, 항구·등대가 장소처럼 보임(컬러 큐브 아님)
