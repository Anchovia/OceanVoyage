# 실행 상태 점검표

이 문서는 OceanVoyage 전환 작업 중 프로젝트가 계속 정상 실행되는지 확인하기 위한 수동 점검표다.

현재 저장소는 Pastel-Farm에서 복제된 초기 상태이므로, 아직 농장 게임 코드가 남아 있다.

따라서 이 점검표는 “현재 기준으로 정상 동작하는지”를 확인하기 위한 임시 기준이다.

이 문서는 자동 테스트를 대체하지 않는다.

다만 코드 삭제, 이름 변경, 렌더러 정리, GameState 교체 전후에 최소한의 회귀를 확인하기 위해 사용한다.

---

## 1. 기본 빌드 확인

### Debug 빌드

```bash
cmake -B build -S .
cmake --build build --config Debug
```

확인 항목:

- [ ] CMake configure가 성공한다.
- [ ] shader compile 단계가 성공한다.
- [ ] C++ 빌드가 성공한다.
- [ ] 실행 파일이 생성된다.
- [ ] 실행 파일 옆에 `shaders/` 폴더가 복사된다.
- [ ] 실행 파일 옆에 `assets/` 폴더가 복사된다.

---

## 2. 클린 빌드 확인

기존 build 폴더를 지운 뒤 다시 빌드한다.

PowerShell:

```powershell
Remove-Item -Recurse -Force build
cmake -B build -S .
cmake --build build --config Debug
```

Git Bash:

```bash
rm -rf build
cmake -B build -S .
cmake --build build --config Debug
```

확인 항목:

- [ ] clean configure가 성공한다.
- [ ] clean build가 성공한다.
- [ ] FetchContent 의존성 구성이 성공한다.
- [ ] Vulkan SDK의 `glslc`를 찾는다.
- [ ] 모든 shader가 `.spv`로 컴파일된다.

---

## 3. 실행 확인

확인 항목:

- [ ] 프로그램이 실행된다.
- [ ] 창이 생성된다.
- [ ] 첫 화면이 지나치게 오래 흰 화면으로 머물지 않는다.
- [ ] 게임 화면이 표시된다.
- [ ] 마우스 / 키보드 입력이 반응한다.
- [ ] 프레임이 계속 갱신된다.
- [ ] 창 닫기가 정상 동작한다.
- [ ] 종료 시 크래시가 발생하지 않는다.

---

## 4. 렌더링 확인

확인 항목:

- [ ] 지형 또는 월드가 표시된다.
- [ ] 플레이어 또는 임시 오브젝트가 표시된다.
- [ ] 오브젝트가 비정상적으로 검게 보이지 않는다.
- [ ] 텍스처가 지나치게 흐리거나 깨지지 않는다.
- [ ] 그림자가 표시된다.
- [ ] 그림자가 심하게 깜빡이지 않는다.
- [ ] 하늘 / 안개 / 시간대 표현이 정상적으로 보인다.
- [ ] 후처리가 적용된다.
- [ ] UI가 화면 위에 표시된다.

---

## 5. 입력 확인

확인 항목:

- [ ] 이동 키가 동작한다.
- [ ] 마우스 입력이 동작한다.
- [ ] 상호작용 키가 동작한다.
- [ ] UI 조작이 동작한다.
- [ ] 창 focus를 잃었다가 돌아와도 입력이 이상해지지 않는다.

---

## 6. 저장 / 로드 확인

확인 항목:

- [ ] 저장 명령이 동작한다.
- [ ] 저장 파일이 생성된다.
- [ ] 저장 후 게임을 종료해도 크래시가 없다.
- [ ] 재실행 시 저장 데이터가 로드된다.
- [ ] 플레이어 위치가 복원된다.
- [ ] 인벤토리 상태가 복원된다.
- [ ] 바닥 드롭 아이템이 복원된다.
- [ ] 수정된 월드 / 청크 상태가 복원된다.
- [ ] 잘못된 저장 파일이 있을 때 프로그램이 즉시 크래시하지 않는다.

---

## 7. 청크 / 월드 확인

확인 항목:

- [ ] 시작 위치 주변 청크가 로드된다.
- [ ] 이동하면 새 청크가 로드된다.
- [ ] 멀어진 청크가 정리된다.
- [ ] 청크 경계에서 눈에 띄는 깨짐이 없다.
- [ ] 저장 후 다시 불러와도 수정된 청크 상태가 유지된다.

---

## 8. 개발 UI 확인

개발 빌드 옵션이 켜져 있을 때 확인한다.

확인 항목:

- [ ] Dev UI가 표시된다.
- [ ] FPS 또는 frame time 정보가 표시된다.
- [ ] GPU profiling 정보가 표시된다.
- [ ] AA 옵션이 변경된다.
- [ ] 디버그 옵션 변경 후 크래시가 없다.

---

## 9. 전환 작업 후 필수 확인

다음 작업 이후에는 반드시 이 점검표를 다시 확인한다.

- 프로젝트 이름 변경
- CMake 설정 변경
- shader 목록 변경
- renderer 초기화 코드 변경
- VulkanContext 생성 / 파괴 코드 변경
- GameState 변경
- World / Chunk 변경
- save/load 변경
- 농장 시스템 제거
- 최소 Ocean test scene 추가

---

## 10. 현재 기준 합격 조건

전환 초기 단계에서는 모든 항목이 완벽할 필요는 없다.

다만 다음 항목은 반드시 만족해야 한다.

- [ ] 빌드가 성공한다.
- [ ] 프로그램이 실행된다.
- [ ] 창이 뜬다.
- [ ] 기본 화면이 표시된다.
- [ ] 입력이 완전히 죽지 않는다.
- [ ] 저장 / 로드가 즉시 크래시하지 않는다.
- [ ] 종료 시 크래시가 없다.
